DMA 原理和工作流程
发表于2026-03-16|device
在计算机系统中，数据搬运是一项极其频繁且繁重的任务。如果每一次硬盘读取、网卡收发都要 CPU 亲自去一字节一字节地搬运，哪怕 CPU 算力再强，也会被这些“体力活”彻底拖垮。 为了解放 CPU，DMA（Direct Memory Access，直接内存访问） 技术应运而生。 一、 核心概念：什么是 DMA 与 DMA Buffer？1. DMA（直接内存访问）本质上，DMA 是一种硬件级的数据旁路机制。它的核心是一个名叫 DMAC（DMA控制器） 的硬件（现在通常集成在主板芯片组或外设内部，如 PCIe 设备的 Bus Master）。 简单来说，它的工作就是：“把数据搬运这种繁重的体力活从 CPU 手里接管过来，让 CPU 去干更有价值的计算工作。”在使用 DMA 的场景下，CPU 只需要在开头交代一下任务（源地址、目的地址、传输大小），在结尾接收一下中断（验收成果），中间的数据流转完全由 DMA 硬件独立完成。 2.
