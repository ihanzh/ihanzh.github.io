GPGPU 实验记录

前 12 个实验都比较简单，根据要求配置和读取 GPGPUState 的字段即可

gpgpu_core_exec_kernel

调用者会设置好 kernel_addr，这就是 code 的起始地址，然后后续就需要解析并执行 code，目前所有的 code 都是 riscv 指令，解析并执行的过程和 CPU 的指令执行类似，唯一不同的是 code 中的寄存器是 GPU 寄存器而不是 CPU 寄存器。也就是每个 lane 的寄存器文件。

其中 RV32I 的指令解析之前在 nemu 中做过，直接参考这个框架让 ai 写了。其主要的逻辑还是 switch case 来解析指令，然后根据指令类型来执行指令。

然后浮点数的解析也类似，有一个点就是 GPGPULane 结构体里面存储的 fpr 是 uint32_t 类型，所以需要先把它转换成 float 类型才能进行浮点数的运算，转换的方法是通过 union 来实现的。

static inline float uint32_to_float(uint32_t value)
{
    union {
        uint32_t u;
        float f;
    } conv = { .u = value };
    return conv.f;
}

static inline uint32_t float_to_uint32(float value)
{
    union {
        float f;
        uint32_t u;
    } conv = { .f = value };
    return conv.u;
}

最后一个实验需要实现低精度浮点转换扩展指令，BF16 和 FP8 的转换方法 softfloat 中提供了实现，直接使用了 softfloat 的函数来实现 BF16 和 FP8 的转换。

最后一个 float32_to_float4_e2m1 函数没有实现，需要自己进行实现，E2M1 的编码方式如下：

编码 (3bit) 值
000 0
001 0.5
010 1.0
011 1.5
100 2.0
101 3.0
110 4.0
111 6.0

直接按数据绝对值中间值划分编码即可：

if (af < 0.25f) {
    bits = 0x0u; /* zero */
} else if (af < 0.75f) {
    bits = 0x1u; /* 0.5 */
} else if (af < 1.25f) {
    bits = 0x2u; /* 1.0 */
} else if (af < 1.75f) {
    bits = 0x3u; /* 1.5 */
} else if (af < 2.5f) {
    bits = 0x4u; /* 2.0 */
} else if (af < 3.5f) {
    bits = 0x5u; /* 3.0 */
} else if (af < 5.0f) {
    bits = 0x6u; /* 4.0 */
} else {
    bits = 0x7u; /* 6.0 saturate */
}
