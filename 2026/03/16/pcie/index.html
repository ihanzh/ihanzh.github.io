PCIe 原理和资源访问

发表于2026-03-16|更新于2026-04-15|device
|浏览量:

本文是对 PCIe 原理和资源访问 视频内容的整理和总结。

一、 PCIe 基本原理与通信机制

PCIe 的设计与早期的 PCI 总线有着本质的区别。

  • 拓扑结构差异：传统的 PCI 采用的是总线拓扑，设备之间需要进行总线仲裁，并且共享带宽 。而 PCIe 采用的是树形拓扑，实现了端到端的通信，每个设备都能独享链路带宽 。

  • 物理层通信：PCIe 的两个端点通过双 Link 进行全双工传输 。其中一个 Link 负责发送（Tx），另一个负责接收（Rx），这两个 Link 共同组成了一个 Lane（通道） 。

  • 灵活的带宽配置：一条 PCIe 传输链路可以由多个 Lane 组成，系统支持 1、2、4、8、16 甚至 32 个 Lane，以此来灵活匹配不同设备对带宽的需求 。

二、 PCIe 核心组件架构

在 PCIe 的树形拓扑中，有三个核心组件构成了整个网络 ：

  1. Root Complex (RC)：这是连接 CPU、内存与 PCIe 树的“根节点”，它不仅负责 CPU/内存与 PCIe 设备之间的数据传输，还处理设备间的 P2P 通信，并实现 IOMMU 功能 。

  2. Bridge / Switch：桥接设备。它包含了内置在 RC 中的 Root Port、上游端口（Upstream Port）和下游端口（Downstream Port） 。Switch 能够极大地扩展 PCIe 接口的数量。

  1. Endpoint (EP)：端点设备，也就是我们实际插在主板上的 PCIe 卡（如显卡、网卡等）。
* PCIe Endpoint：访问地址空间>=4G memory，支持32bit和64 bit；无法产生I/O请求。
* Legacy Endpoint：访问地址空间<4G memory，支持32bit；可以产生I/O请求。
* Root Complex Integrated Endpoint：直接集成到Root Complex的设备，不需要通过Root Port连接。比如CPU的集成显卡。

💡 实用技巧：在 Linux 系统中，我们可以通过 lspci 命令来查看 PCIe 的拓扑结构和设备详细配置（例如 lspci -tv 查看树形结构，lspci -vvv 查看详细信息） 。

三、 数据传输的灵魂：TLP (Transaction Layer Packet)

PCIe 的硬件数据传输在逻辑上分为三层：事务层、数据链路层和物理层 。其中，对软件开发者最重要的一层是事务层。

  • 事务层通过 TLP（事务层数据包） 来实现数据的路由和寻址，这就好比 TCP/IP 网络中的 IP 数据包 。

  • TLP 的请求类型主要分为四种：Memory（内存读写）、I/O（IO 读写）、Configuration（配置空间读写）和 Message（消息机制） 。

  • 路由方式：为了将 TLP 准确送达目标设备，中间的桥设备会根据三种路由规则进行转发：Address Routing（基于地址，用于 Memory/IO）、ID Routing（基于 BDF 标识，用于配置空间读写）以及 Implicit Routing（隐含路由，用于消息） 。

Address Routing 设备会判断地址是否落在自己的 Bar 范围内，如果是就处理请求，桥设备也有自己的路由表，记录了下游设备的地址范围，来决定是否转发 TLP 。

同样，ID Routing 设备会根据 TLP 中的 BDF 标识来判断是否处理请求或转发 。桥设备的路由表中会记录下游设备的 Bus 号范围，来决定是否转发 TLP 。

四、 PCIe 资源分配与访问

系统启动时，BIOS 或操作系统会枚举所有 PCIe 设备，并为它们分配必需的系统资源 。

1. BDF 标识号

每个 PCIe 设备在系统中都有一个唯一的“身份证”—— BDF (Bus:Device.Function) 。

  • Bus：8 位，最多支持 256 条总线 。

bus 号由桥设备分配，根桥（Root Port）分配给下游设备的 bus 号范围通常是连续的 。

  • Device：5 位，每个 bus 最多连接 32 个设备 。

  • Function：3 位，每个物理设备最多可包含 8 个独立的功能接口（当设备为多功能设备时，此值非 0） 。

BDF 实际不存在于硬件和配置空间中，而是由系统软件在枚举过程中动态分配的逻辑标识符，用于软件层面识别和访问设备 。

2. Configuration Space (配置空间)

配置空间是软件与设备交互的控制面板。每个设备的 Function 都有一个独立的配置空间，包含了设备的基本信息、状态寄存器以及资源需求等 。

配置空间包括

  • 头部：64 字节

  • Capabilities（可选）：192字节，一组拓展寄存器，每个功能接口可以有一个链表结构的能力列表，记录了设备支持的扩展功能（如 MSI、MSI-X、Power Management 等） 。

  • PCIe 拓展空间：4096-256字节

  • PCI 时代配置空间只有 256 字节，而在 PCIe 中被扩展到了 4KB 。

  • 访问方式有传统的 IO 端口访问（0xCF8 和 0xCFC）以及更现代的 MMIO（内存映射 IO） 方式 。

  • 端点设备使用的是 Type 0 头结构，而桥设备使用的是 Type 1 头结构 。

3. BAR (Base Address Register)

BAR 是极为关键的寄存器。设备的内部 RAM 或寄存器要想被 CPU（Host）直接读写，必须被统一映射到 Host 的物理地址空间中 。

  • 系统软件通过读取 BAR 来了解设备需要多大的地址空间，并为其分配物理基地址 。然后将基地址写入 BAR 寄存器中，Host 就可以通过基地址加上偏移量的方式来访问设备内存了 。

  • 在 Linux 中，内核需要通过 ioremap 将这些物理地址映射到虚拟地址空间后才能安全访问 。

  • 应用程序需要通过 mmap 设备的 resource 文件，映射到用户空间后才能访问设备资源 。

五、 DMA 与中断机制

  • DMA (Direct Memory Access)：这允许设备内部的 DMA 控制器主动发送 Memory Write TLP，将数据直接搬运到系统内存，全程无需 CPU 干预，极大地释放了 CPU 算力 。

  • 中断机制：当设备需要通知 CPU 时，会触发中断。PCIe 必须支持现代的 MSI (Message Signaled Interrupt) 或 MSI-X 中断 。

  • 它们本质上是设备向特定地址（如 0xFEE00000）发送的一个带数据的 Memory Write TLP 。

  • MSI 最多支持 32 个中断向量，而 MSI-X 则通过内存中的 Table 结构，支持多达 2048 个中断，并允许中断向量非连续分配 。
