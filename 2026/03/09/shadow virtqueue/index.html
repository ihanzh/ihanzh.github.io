Shadow Virtqueue（SVQ）是 QEMU vhost-vdpa 中的一个关键机制，用于在 Guest 和 vDPA 后端硬件之间插入一个”影子”层，实现地址翻译、流量拦截与回调处理。本文梳理 SVQ 的整体架构、文件描述符体系以及各方向的通知链路。

一、整体架构

SVQ 将原本 Guest 直通 Backend 的 virtqueue 路径拦截，在中间插入 QEMU 作为代理：

SVQ 维护四个文件描述符，分别对应两侧的 kick/call：

FD 方向 说明
svq->svq_kick Guest → QEMU 接管原有 Guest kick fd，QEMU 监听
svq->hdev_kick QEMU → Backend QEMU 新建，用于通知 Backend
svq->hdev_call Backend → QEMU QEMU 新建，Backend 完成时触发
svq->svq_call QEMU → Guest 保留原有 call fd，QEMU 发起中断
