RoPE（旋转位置编码）

发表于2026-05-09|更新于2026-05-12|LLM inference
|浏览量:

RoPE（Rotary Position Embedding，旋转位置编码）是现代大语言模型（LLM）中一种非常流行的位置编码方案，其通过把位置信息"旋转"进 Q 和 K 向量中，使得两个位置 m 和 n 的向量点积自然包含它们的相对距离 (m-n)。

Step 1：计算 head_dim 维度上的旋转频率 inv_freq

RoPE 的第一步是为 Q/K 向量的每个维度对分配一个旋转频率。

以 head_dim=128 为例，我们把 128 维向量分成 64 对，每对使用不同频率的 θ：

θ₀ = 1 / base^(0/128)    → 频率最高，旋转最快（捕捉近距离关系）
θ₁ = 1 / base^(2/128)    →
θ₂ = 1 / base^(4/128)    →
  ...
θ₆₃ = 1 / base^(126/128)  → 频率最低，旋转最慢（捕捉远距离关系）

直觉：类似钟表——秒针转得快（高频，感知短间隔），时针转得慢（低频，感知长间隔）。64 个频率组合在一起，可以精确编码任意位置。

inv_freq 的大小：（head_dim/2,）= (64,)：

其中 base 通常是 10000（原始 Transformer）或 1000000（Qwen3 长上下文）。base 越大，低频成分变化越慢，能编码的最大距离越远。

Step 2：乘以位置 ID，得到每个位置的旋转角度

将步骤 1 的频率向量与位置索引 [0, 1, 2, ..., seq_len-1] 做外积，得到每个位置在每个维度对上的旋转角度：

                        inv_freq (64 个频率)
                    θ₀      θ₁      θ₂     ...   θ₆₃
位置 0:          0·θ₀    0·θ₁    0·θ₂    ...   0·θ₆₃     ← 不旋转
位置 1:          1·θ₀    1·θ₁    1·θ₂    ...   1·θ₆₃     ← 旋转一点
位置 2:          2·θ₀    2·θ₁    2·θ₂    ...   2·θ₆₃     ← 旋转更多
  ...
位置 m:          m·θ₀    m·θ₁    m·θ₂    ...   m·θ₆₃

得到的 freqs 矩阵形状为 (seq_len, head_dim/2)，每行对应一个位置的旋转角度。

class RotaryEmbedding(nn.Module):
    def __init__(self, head_dim: int, base: float = 1000000.0):
        super().__init__()
        # inv_freq: (head_dim // 2,) ，每个维度对的旋转频率 θ
        inv_freq = 1.0 / (base ** (torch.arange(0, head_dim, 2, dtype=torch.float32) / head_dim))
        self.register_buffer("inv_freq", inv_freq, persistent=False)

# 这个只适用于一个 batch 内所有的样本的 position_ids 都相同的情况，因为它只使用了 position_ids[0] 来计算位置编码，如果不同样本的 position_ids 不同，那么这个实现就不适用了，需要修改代码来支持不同样本的 position_ids
    def forward(self, position_ids: torch.Tensor) -> tuple[torch.Tensor, torch.Tensor]:
        # position_ids: (batch_size, seq_len)
        # position_ids[0].float() -> (seq_len,)，取出第一个样本的 position_ids，并转换为 float32 类型
        # self.inv_freq: (head_dim // 2,)
        # torch.outer(position_ids[0].float(), self.inv_freq) -> (seq_len, head_dim // 2)
        freqs = torch.outer(position_ids[0].float(), self.inv_freq);

Step 3：计算 cos 和 sin 值

将角度矩阵复制一份（匹配完整的 head_dim），然后计算 cos 和 sin：

class RotaryEmbedding(nn.Module):
    def __init__(self, head_dim: int, base: float = 1000000.0):
        ...

    def forward(self, position_ids: torch.Tensor) -> tuple[torch.Tensor, torch.Tensor]:
        ...
        # emb: (seq_len, head_dim) 的张量
        # emb.cos().unsqueeze(0) -> (1, seq_len, head_dim)，计算位置编码的余弦部分，并在第0维前面添加一个维度
        # cos -> cos(m·θ_i) 矩阵, sin -> sin(m·θ_i) 矩阵
        emb = torch.cat((freqs, freqs), dim=-1)
        return emb.cos().unsqueeze(0), emb.sin().unsqueeze(0)

cos 和 sin 的形状都是 (seq_len, head_dim)，每个位置的每个维度都有对应的 cos/sin 值。

为什么要 cat 复制？ 因为每对维度共享同一个频率。head_dim=128 分成 64 对，每对 (x₀, x₁) 用同一个 θ，所以 cos/sin 需要扩展为 128 维来逐元素操作. 并且这样拓展后，前后半配对——将向量分成前半 x[:64] 和后半 x[64:]，配对为 (0,64), (1,65), (2,66), …

这样做是为了利用连续内存访问，避免交错索引，两种配对方式数学上完全等价，只是维度排列不同。

Step 4：旋转矩阵乘法

有了每个位置的 cos/sin 值，就可以对 Q/K 向量做旋转了。

2D 旋转的数学原理：

对于一对维度 (x₀, x₁)，位置 m 的旋转矩阵为：

[cos(mθ)  -sin(mθ)]   [x₀]   [x₀·cos(mθ) - x₁·sin(mθ)]
[sin(mθ)   cos(mθ)] × [x₁] = [x₀·sin(mθ) + x₁·cos(mθ)]

对于高维向量（head_dim=128），64 对维度各自独立旋转（互不干扰），相当于一个分块对角矩阵。

# 90 顺时针旋转：R(90°) = [[0, -1], [1, 0]]，即 (x₀, x₁) * R(90°) → (-x₁, x₀)
def rotate_half(x: torch.Tensor) -> torch.Tensor:
    # x: (batch_size, num_heads, seq_len, head_dim)
    # x1: (batch_size, num_heads, seq_len, head_dim // 2)，取出前半部分
    # x2: (batch_size, num_heads, seq_len, head_dim // 2)，取出后半部分
    # -x2: (batch_size, num_heads, seq_len, head_dim // 2)，对后半部分取反
    # torch.cat((-x2, x1), dim=-1)  将 -x2 和 x1 在最后一个维度上拼接起来，得到旋转后的张量
    # (batch_size, num_heads, seq_len, head_dim)
    x1 = x[..., :x.shape[-1] // 2]
    x2 = x[..., x.shape[-1] // 2:]
    return torch.cat((-x2, x1), dim=-1)

def apply_rotary_pos_emb(q: torch.Tensor, k: torch.Tensor, cos: torch.Tensor, sin: torch.Tensor) -> tuple[torch.Tensor, torch.Tensor]:
    # cos/sin: (1, seq_len, head_dim) -> (1, 1, seq_len, head_dim)
    cos = cos.unsqueeze(1)
    sin = sin.unsqueeze(1)
    # q (x₀, x₁)
    # rotate_half(q) -> (-x₁, x₀)
    # q_embed = q * cos + rotate_half(q) * sin -> (x₀·cos + (-x₁)·sin, x₁·cos + x₀·sin) -> 正好是旋转矩阵乘法的结果
    q_embed = q * cos + rotate_half(q) * sin
    k_embed = k * cos + rotate_half(k) * sin

    return q_embed, k_embed

cos, sin = position_embedding
q, k = apply_rotary_pos_emb(q, k, cos, sin)

Step 5：应用到 Q 和 K

在 Attention 计算中，RoPE 只应用于 Q 和 K，不应用于 V：

Q_proj ──► Q ──► 应用 RoPE ──┐
                              ├──► Q·K^T ──► Attention Score ──► ...
K_proj ──► K ──► 应用 RoPE ──┘

V_proj ──► V ──────────────────────────────────────────────► ...
                           （V 不需要 RoPE）

为什么 RoPE 能编码相对位置？

当计算位置 m 的 Q 和位置 n 的 K 的点积时：

Q_m · K_n = (R(mθ) · q) · (R(nθ) · k)
          = q · R((m-n)θ) · k    ← 只依赖相对距离 (m-n)！

旋转矩阵 R 具有正交性：$R(a)^T \cdot R(b) = R(b-a)$。所以 attention score 自然包含了两个 token 之间的相对距离，不需要额外计算。

RoPE 总结

特性 说明
相对位置 Q·K 点积自然包含相对距离，无需显式计算
外推能力 理论上可处理训练时未见过的长度
零参数量 inv_freq 是固定公式计算的，不需要学习
计算高效 只需简单的逐元素乘法和加法
兼容性好 可与 Flash Attention 等优化技术结合

长上下文扩展：

Qwen3 使用 base=1000000（而非原始的 10000），使低频成分变化更慢，从而支持更长的上下文（40K+ tokens）。这种技术称为 NTK-aware scaling 或 Dynamic NTK。
