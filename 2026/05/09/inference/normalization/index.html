Normalization（归一化）

发表于2026-05-09|更新于2026-05-12|LLM inference
|浏览量:

在现代大语言模型（LLM）中，Normalization（归一化） 是一个至关重要的组件，主要负责稳定训练过程、加速收敛、提升模型性能。它的核心思想是对输入数据进行标准化处理，使其具有统一的分布特征，从而帮助模型更好地学习和泛化。

LayerNorm 详解

LayerNorm（Layer Normalization）是 Transformer 模型中最常用的归一化方法。它将输入的每个样本（通常是一个向量）进行归一化处理，使其均值为0，方差为1，并通过可学习的参数进行缩放和平移。

核心公式：

对于输入向量 $x = [x_1, x_2, …, x_d]$（d 是 hidden_size）：

$$
\text{LayerNorm}(x) = \gamma \cdot \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta
$$

其中：

  • $\mu = \frac{1}{d}\sum_{i=1}^{d} x_i$（均值）
  • $\sigma^2 = \frac{1}{d}\sum_{i=1}^{d} (x_i - \mu)^2$（方差）
  • $\gamma$（scale）和 $\beta$（shift）是可学习参数
  • $\epsilon$ 是防止除零的小常数（如 1e-6）

参数量：2 × hidden_size（gamma 和 beta 各 hidden_size 个）

为什么 γ 和 β 很重要？

如果只做归一化（强制均值=0，方差=1），会限制网络的表达能力。通过可学习的 γ 和 β：

  • 网络可以"学习"恢复原始分布（如果需要的话）
  • 当 γ=σ, β=μ 时，相当于恒等变换（什么都不做）
  • 网络可以在"归一化"和"保持原样"之间自由选择

RMSNorm：更简单的替代方案

现代 LLM（如 Llama、Qwen、Mistral）普遍使用 RMSNorm 替代 LayerNorm。

核心公式：

$$
\text{RMSNorm}(x) = \gamma \cdot \frac{x}{\text{RMS}(x)}
$$

其中：

$$
\text{RMS}(x) = \sqrt{\frac{1}{d}\sum_{i=1}^{d} x_i^2 + \epsilon}
$$

class RMSNorm(nn.Module):
    def __init__(self, hidden_size: int, eps: float = 1e-6):
        super().__init__()
        # torch.ones(hidden_size) -> (hidden_size,) 的张量，所有元素都是 1
        # nn.Parameter() 表示 self.weight 也是一组权重，需要从模型的预训练权重中加载
        self.weight = nn.Parameter(torch.ones(hidden_size))
        self.eps = eps

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # x: (batch_size, seq_len, hidden_size) x.float() 将输入张量转换为 float32 类型，确保计算的数值稳定性
        # x.pow(2).mean(-1, keepdim=True) 计算最后一维度张量的平方的均值，，并保持维度不变 -> (batch_size, seq_len, 1)
        # torch.rsqrt() 计算输入张量的平方根的倒数，即 1 / sqrt(x)，在 rsqrt 之前加上 eps 防止除以零 -> (batch_size, seq_len, 1)
        x_float = x.float()
        rms = torch.rsqrt(x_float.pow(2).mean(-1, keepdim=True) + self.eps)

        # x_float: (batch_size, seq_len, hidden_size)
        # rms: (batch_size, seq_len, 1)
        # x_float * rms: (batch_size, seq_len, hidden_size)，最后一个维度的每个元素都乘以 rms 值，实现了归一化的效果
        # self.weight: (hidden_size,) 的可训练参数，广播机制会自动将其扩展到 (batch_size, seq_len, hidden_size)
        # 每个元素都乘以对应的权重值，这个权重可以调整归一化后的输出的尺度，允许模型在训练过程中学习到合适的缩放方式
        # 最终输出 -> (batch_size, seq_len, hidden_size)
        return (x_float * rms).to(x.dtype) * self.weight

关键区别：

特性 LayerNorm RMSNorm
减均值（中心化） ✅ 是 ❌ 否
除标准差 ✅ 是 ❌ 否（除 RMS）
加偏置 β ✅ 是 ❌ 否
参数量 2d d
计算量 较多 较少

为什么 RMSNorm 有效：

研究表明，LayerNorm 的主要作用来自缩放（除以某个统计量），而不是中心化（减均值）。RMSNorm 去掉了中心化步骤，但保留了核心的缩放作用，同时：

  • 减少 ~50% 的计算量
  • 减少 50% 的参数量
  • 实验效果相当甚至更好

参数量：hidden_size（只有 gamma，没有 beta）
