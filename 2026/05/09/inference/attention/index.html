Attention（注意力机制）

发表于2026-05-09|更新于2026-05-11|LLM
|浏览量:

在现代大语言模型（LLM）中，Attention（注意力机制） 是一个核心组件，负责捕捉输入序列中不同位置之间的依赖关系。它通过计算查询（Query）、键（Key）和值（Value）之间的相似度来动态地调整信息流，使得模型能够更有效地理解和生成文本。

模型中有三个重要的矩阵：

  • $W_Q$：将输入投影到查询空间的权重矩阵：（hidden_size, hidden_size）
  • $W_K$：将输入投影到键空间的权重矩阵：（hidden_size, hidden_size）
  • $W_V$：将输入投影到值空间的权重矩阵：（hidden_size, hidden_size）

Q、K、V 的直观理解

通过对输入序列进行线性变换，我们得到三个矩阵：

输入 $X$ 的形状为（batch, seq_len, hidden_size）：

  • $Q = X W_Q$：查询矩阵，形状为（batch, seq_len, hidden_size）
  • $K = X W_K$：键矩阵，形状为（batch, seq_len, hidden_size）
  • $V = X W_V$：值矩阵，形状为（batch, seq_len, hidden_size）

Attention 的核心思想：对于每个查询（Query），在所有键值对（Key-Value）中找到相关的内容，然后加权汇总。

把 Attention 想象成一个信息检索系统：

你在图书馆找书
     │
     ▼
Query (Q): 你的问题/需求
     "我想找关于 Python 的入门书"
     │
     ▼
Key (K): 每本书的索引/标签
     ["Python入门", "Java高级", "数据结构", ...]
     │
     ▼
匹配程度 = Q · K^T
     [0.95, 0.1, 0.3, ...]  ← 相关性分数
     │
     ▼
Softmax → 注意力权重
     [0.7, 0.01, 0.15, ...]  ← 归一化后
     │
     ▼
Value (V): 每本书的实际内容
     取出相关书籍，根据相关性加权组合

Self-Attention（自注意力）：

在 Transformer 中，Q、K、V 都来自同一个输入序列（即将输入序列做不同的线性变换），因此称为 Self-Attention。每个位置的查询向量 Q[i] 会与所有位置的键向量 K[j] 进行点积，得到一个分数，表示位置 i 对位置 j 的注意力程度。然后通过 Softmax 归一化这些分数，得到注意力权重，最后用这些权重对值矩阵 V 进行加权求和，得到输出。

# 输入: "The cat sat on the mat"
# 每个词都生成自己的 Q, K, V

# 当处理 "sat" 这个词时:
Q_sat = "sat 想要查找什么信息？"
K_*   = "所有词在'被查找'时的表示"
V_*   = "所有词的实际语义内容"

# sat 的输出 = 加权组合所有词的 V
#   如果 sat 与 cat 相关，cat 的 V 权重会较高

Attention 完整计算流程

   Q        K^T           Softmax          V
(seq, d) × (d, seq)  →  (seq, seq)  ×  (seq, d)  →  (seq, d)
   │          │            │             │
   └────┬─────┘            │             │
        ▼                  ▼             │
     scores             weights          │
   (seq, seq)         (seq, seq)         │
        │                  │             │
        └──────────────────┬─────────────┘
                           ▼
                         output
                        (seq, d)

Step 1: 计算注意力分数（Scores）

$$
\text{scores} = QK^T
$$

Q: (batch, seq_len, hidden_size)

K: (batch, seq_len, hidden_size) → $K^T$: (batch, hidden_size, seq_len)

scores: (batch, seq_len, seq_len) → 每个位置对所有位置的相似度分数

本质是对每个位置的查询向量 Q[i]，与所有位置的键向量 K[j] 进行点积，得到一个分数，表示位置 i 对位置 j 的注意力程度。

Step 2: 缩放（Scaling）

$$
\text{ScaledScores} = \frac{QK^T}{\sqrt{d_k}}
$$

为什么要除以 $\sqrt{d_k}$？

  • 在 Attention 中，点积 $Q_i \cdot K_j$ 是两个向量的累加和。
  • 如果 $d_k$ 很大，这个和的绝对值也会变大，Softmax 的输入就会变得非常极端。
  • 极端输入会让 Softmax 变得“尖锐”，注意力权重集中在少数位置上，训练时梯度变得很小。

更直观地看：

  • 假设 $Q$ 和 $K$ 的每个元素都是独立的标准正态分布 $\mathcal{N}(0,1)$。
  • 那么每一项 $Q[i] \times K[i]$ 的期望为 0，方差为 1。
  • 点积
    $$Q \cdot K = \sum_{i=1}^{d_k} Q[i] K[i]$$
    相当于 $d_k$ 个独立随机变量的和。
  • 这些和的方差就是 $d_k$，因此点积的标准差约为 $\sqrt{d_k}$。

举例：

  • 当 $d_k=128$ 时，标准差约为 $\sqrt{128}\approx 11$。
  • 这意味着点积值常常会落在大约 $[-20, 20]$ 的范围内。
  • 如果直接用这个值做 Softmax，输出就会非常尖锐，注意力几乎只落在一个位置上。

把点积除以 $\sqrt{d_k}$ 之后：

  • 结果的标准差变成约 1。
  • Softmax 的输入不再随维度增长而放大。
  • 注意力权重更平滑，训练更稳定。

Step 3: 应用掩码（Masking）

对于因果语言模型（如 GPT），位置 t 不能看到 t+1, t+2, … 的信息，所以需要一个掩码矩阵，将未来位置的分数设置为负无穷（-∞），使得 Softmax 后这些位置的权重变为 0：

$$
\text{MaskedScores}{i,j} = \begin{cases}
\text{ScaledScores}{i,j}, & \text{if } j \leq i \
-\infty, & \text{if } j > i
\end{cases}
$$

Step 4: Softmax 归一化

在每个位置 i 上，对所有位置 j 的分数进行 Softmax 归一化，得到注意力权重：

$$
\text{Weights} = \text{softmax}(\text{ScaledScores})
$$

例如：

输入句子: "The cat sat on the mat"

Attention 权重矩阵（某一层某一头）:

          The   cat   sat   on   the   mat
    The  [0.9   0.05  0.02  0.01  0.01  0.01]
    cat  [0.3   0.5   0.1   0.05  0.03  0.02]
    sat  [0.1   0.4   0.3   0.1   0.05  0.05]  ← sat 主要关注 cat
    on   [0.1   0.1   0.2   0.4   0.1   0.1 ]
    the  [0.05  0.1   0.1   0.1   0.3   0.35]
    mat  [0.05  0.2   0.1   0.1   0.15  0.4 ]  ← mat 关注自己和 cat

Step 5: 权重与值矩阵 V 相乘

最后，将注意力权重与值矩阵 V 相乘，V 代表每个位置的实际价值：

$$
\text{Output} = \text{Weights} \cdot V
$$

  • Weights: (batch, seq_len, seq_len)

  • V: (batch, seq_len, hidden_size)

  • Output: (batch, seq_len, hidden_size)

class Qwen3Attention(nn.Module):
    def __init__(self, config, layer_idx: int):
        super().__init__()

        self.num_heads = config.num_attention_heads
        self.num_kv_heads = config.num_key_value_heads
        self.num_kv_groups = self.num_heads // self.num_kv_heads
        self.head_dim = config.head_dim
        # sqrt(d_k) 是缩放因子
        self.scaling = self.head_dim ** -0.5

        # q_proj: (hidden_size, num_heads * head_dim)
        # k_proj: (hidden_size, num_kv_heads * head_dim)
        # v_proj: (hidden_size, num_kv_heads * head_dim)
        # o_proj: (num_heads * head_dim, hidden_size)
        self.q_proj = nn.Linear(config.hidden_size, self.num_heads * self.head_dim, bias = False)
        self.k_proj = nn.Linear(config.hidden_size, self.num_kv_heads * self.head_dim, bias = False)
        self.v_proj = nn.Linear(config.hidden_size, self.num_kv_heads * self.head_dim, bias = False)
        self.o_proj = nn.Linear(self.num_heads * self.head_dim, config.hidden_size, bias=False)

        # q/k 的最后一维是 head_dim，所以这里的 RMSNorm 也应当按 head_dim 归一化。
        self.q_norm = RMSNorm(self.head_dim, config.rms_norm_eps)
        self.k_norm = RMSNorm(self.head_dim, config.rms_norm_eps)

    def forward(self, hidden_states: torch.Tensor, attention_mask, position_embedding) -> torch.Tensor:
        B, S, _ = hidden_states.shape

        # q/k/v 都走下述流程
        # hidden_states: (batch_size, seq_len, hidden_size)
        # q_proj: (hidden_size, num_heads * head_dim)
        # hidden_states @ q_proj: (batch_size, seq_len, num_heads * head_dim)
        # reshape: (batch_size, seq_len, num_heads, head_dim)
        # transpose: (batch_size, num_heads, seq_len, head_dim)
        # 最后 q -> (batch_size, num_heads, seq_len, head_dim)
        # k/v -> (batch_size, num_kv_heads, seq_len, head_dim)
        q = self.q_proj(hidden_states).view(B, S, self.num_heads, self.head_dim).transpose(1, 2)
        k = self.k_proj(hidden_states).view(B, S, self.num_kv_heads, self.head_dim).transpose(1, 2)
        v = self.v_proj(hidden_states).view(B, S, self.num_kv_heads, self.head_dim).transpose(1, 2)

        # q 和 k 先做 RMSNorm 归一化，再应用 RoPE 位置编码
        q = self.q_norm(q)
        k = self.k_norm(k)

        # cos/sin: (1, seq_len, head_dim)
        cos, sin = position_embedding
        q, k = apply_rotary_pos_emb(q, k, cos, sin)

        # 由于 num_kv_groups = num_heads // num_kv_heads
        # repeat_kv 函数会将 k 和 v 沿着头的维度重复 num_kv_groups 次，
        # 使得 k/v (batch_size, num_kv_heads, seq_len, head_dim) -> (batch_size, num_heads, seq_len, head_dim)
        # 每 num_kv_groups 个 Q 共享一组 KV。
        k = repeat_kv(k, self.num_kv_groups)
        v = repeat_kv(v, self.num_kv_groups)

        # atten_weights: (batch_size, num_heads, seq_len, head_dim) @ (batch_size, num_heads, head_dim, seq_len) -> (batch_size, num_heads, seq_len, seq_len)
        atten_weights = torch.matmul(q, k.transpose(-2, -1)) * self.scaling
        atten_weights = atten_weights + attention_mask
        atten_weights = F.softmax(atten_weights, dim=-1, dtype=torch.float32).to(q.dtype)

        # atten_output: (batch_size, num_heads, seq_len, seq_len) @ (batch_size, num_heads, seq_len, head_dim) -> (batch_size, num_heads, seq_len, head_dim)
        # transpose: (batch_size, seq_len, num_heads, head_dim)
        # reshape: (batch_size, seq_len, num_heads * head_dim)
        atten_output = torch.matmul(atten_weights, v)
        atten_output = atten_output.transpose(1, 2).reshape(B, S, -1)

        # 最后通过 o_proj 线性变换回 hidden_size 维度
        return self.o_proj(atten_output)

多头注意力（Multi-Head Attention）

实际上我们会把 hidden_size 拆成多个"头"（num_heads），每个头的维度是 head_dim = hidden_size / num_heads。每个头都有自己独立的 Q、K、V 权重矩阵，独立计算注意力，最后把所有头的输出拼接起来，再通过一个线性变换得到最终输出。

Q 实际形状: (batch, seq_len, hidden_size)
   ──reshape──► (batch, seq_len, num_heads, head_dim)
   ──transpose──► (batch, num_heads, seq_len, head_dim)
K 和 V 同理

后续 $QK^T$ 的计算会在每个头内独立进行，得到每个头的注意力输出

  • $QK^T$ → (batch, num_heads, seq_len, seq_len)
  • Attention 输出 → (batch, num_heads, seq_len, head_dim)

最后会把所有头的输出拼接起来，得到 (batch, seq_len, hidden_size)

Grouped Query Attention (GQA)

现代 LLM（如 Llama2-70B、Qwen3）使用 GQA 来减少 KV cache 大小：

MHA:  32 个 Q heads, 32 个 K heads, 32 个 V heads
MQA:  32 个 Q heads,  1 个 K head,   1 个 V head
GQA:  32 个 Q heads,  8 个 K heads,  8 个 V heads（Qwen3-8B）
                      ↑
              每 4 个 Q heads 共享一组 KV

例如 MQA 中

  • $W_Q$: (hidden_size, num_heads × head_dim) = (4096, 32 × 128)
  • $W_K$: (hidden_size, num_kv_heads × head_dim) = (4096, 1 × 128)
  • $W_V$: (hidden_size, num_kv_heads × head_dim) = (4096, 1 × 128)

从而计算出来的 Q/K/V 的形状分别是：

  • Q: (batch, seq_len, num_heads × head_dim) = (batch, seq_len, 4096)
  • K: (batch, seq_len, num_kv_heads × head_dim) = (batch, seq_len, 128)
  • V: (batch, seq_len, num_kv_heads × head_dim) = (batch, seq_len, 128)

reshape 和 transpose 后：

  • Q: (batch, num_heads, seq_len, head_dim) = (batch, 32, seq_len, 128)
  • K: (batch, num_kv_heads, seq_len, head_dim) = (batch, 1, seq_len, 128)
  • V: (batch, num_kv_heads, seq_len, head_dim) = (batch, 1, seq_len, 128)

因此所有的 32 个 Q heads 共享同一组 K 和 V

同理 GQA 中每 4 个 Q heads 共享一组 KV，减少了 KV cache 的大小，同时保持了较好的性能。

参数量：

  • $W_Q$: hidden_size × (num_heads × head_dim) = 4096 × 4096 ≈ 16.8M
  • $W_K$: hidden_size × (num_kv_heads × head_dim) = 4096 × 1024 ≈ 4.2M
  • $W_V$: hidden_size × (num_kv_heads × head_dim) = 4096 × 1024 ≈ 4.2M
  • $O$ (输出投影): 4096 × 4096 ≈ 16.8M
  • 每层 Attention 总计：约 42M 参数
